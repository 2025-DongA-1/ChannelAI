# Meta 비즈니스 인증 - 학생 프로젝트 가이드

## 🎓 학생 프로젝트 상황

**현재 상태:**
- 스마트인재개발원 학생
- 동아일보 협업 AI 훈련 과정
- 팀 프로젝트 (상용화 전)

---

## 선택지별 분석

### 1. 법인 ❌
**정의:** 상장 기업
**필요:** 법인 등기부, 주주 명부
**적합성:** 불가능 (법인 아님)

### 2. 개인 사업 ⚠️
**정의:** 1인 운영 사업
**필요:** 개인 사업자 등록증
**적합성:** 사업자 등록 필요

### 3. 파트너십 ⚠️
**정의:** 2인 이상 공동 운영
**필요:** 동업 계약서, 각자 사업자 등록
**적합성:** 팀 프로젝트지만 사업자 필요

### 4. 비상장기업 ❌
**정의:** 유한책임회사(LLC) 등
**필요:** 법인 등록
**적합성:** 법인 아님

### 5. 기관 🟡 (교육 기관 협조 시)
**정의:** 교육 기관, 비영리 단체
**필요:** 기관 사업자 정보
**적합성:** 스마트인재개발원 명의로 가능

---

## 🎯 추천 시나리오

### 시나리오 A: 프로젝트 완성 우선 (추천) ⭐

**선택:** 비즈니스 인증 **건너뛰기**

**이유:**
1. 개발 모드로 충분히 개발 가능
2. 본인 Meta 계정으로 테스트 가능
3. 시연/발표에 문제 없음
4. 포트폴리오로 활용 가능

**진행:**
```
✅ 비즈니스 인증 팝업 닫기
✅ localhost OAuth URL만 사용
✅ 본인 광고 계정으로 실제 데이터 테스트
✅ 프로젝트 완성 → 시연/발표
```

**제한 사항:**
- ⚠️ 다른 사용자는 앱 사용 불가
- ⚠️ Standard Access만 가능 (기본 권한)
- ⚠️ 실제 서비스 런칭 불가

**충분한 경우:**
- ✅ 교육 과정 프로젝트
- ✅ 졸업 작품
- ✅ 취업 포트폴리오
- ✅ 기능 시연

---

### 시나리오 B: 교육 기관 활용

**선택:** **5. 기관**

**조건:**
- 스마트인재개발원 담당자 협조 필요

**필요 서류:**
1. 교육 기관 사업자 등록증
2. 담당자/교수 신분증
3. 공식 이메일 (예: project@smart-edu.or.kr)
4. 프로젝트 승인 문서

**진행:**
```
1. 담당 교수/코디네이터에게 요청
   "Meta 앱 비즈니스 인증을 위해 교육 기관 정보가 필요합니다"

2. 기관 정보 받기
   - 사업자 번호
   - 대표자 정보
   - 공식 이메일

3. "기관" 선택 후 정보 입력

4. 서류 업로드 (교육 기관 명의)

5. 프로젝트 설명 추가
   "스마트인재개발원 교육 과정 중 학생 프로젝트입니다"
```

**장점:**
- ✅ Advanced Access 가능
- ✅ 실제 사용자 테스트 가능
- ✅ 정식 서비스처럼 운영 가능

**단점:**
- ⏱️ 기관 협조 시간 소요 (1-2주)
- 📄 서류 준비 복잡
- 🔄 승인 절차 필요

---

### 시나리오 C: 향후 상용화 계획

**선택:** **2. 개인 사업** 또는 **3. 파트너십**

**조건:**
- 개인 사업자 등록 (세무서, 무료)
- 또는 팀원들과 공동 등록

**절차:**
```
1. 세무서 방문
   - 신분증
   - 임대차 계약서 (사무실)

2. 개인 사업자 등록
   - 업종: 소프트웨어 개발/마케팅 서비스
   - 무료, 즉시 발급

3. Meta 비즈니스 인증
   - "개인 사업" 선택
   - 사업자 등록증 업로드

4. 정식 서비스 운영
```

**적합한 경우:**
- 졸업 후 스타트업 창업 계획
- 실제 고객 유치 목표
- 수익 모델 있음

